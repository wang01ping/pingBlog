    # module.exports和exports|| export和export default 区别

    普及基本概念

    Node模块和CommonJS模块

    1、首先得明白CommonJS模块规范和ES6模块规范完全是两种不同的概念
Node应用是由模块组成，采用CommonJS模块规范，根据这个规范，每个文件就是一个模块，变量都是私有的对其他文件不可见,导入必须使用require
2、common.js规范规定每个模块内部，mmodlue变量代表当前模块。这个变量是一个对象，它得export属性（即module.exports）是对外接口。加载某个模块即是。其实是加载该模块的

    # module.exports 和 exports 的区别

    注意区别

    exports是node为了方便而推出来module.exports缩写的，但是它们还是有区别的。 比如：exports不能直接 exports=function 一个值，这样会切断了exports与module.exports的联系

    exports=function(){
    var a="Hello World"
    return a;

    # export 和 export default的区别

    注意区别

    1. export 在一个页面中可以有多个，但是export default却只有一个
    2. export default 的对象、变量 、函数、类可以没有名字，但是export必须有名字
    3. export 对应的导入是必须是import {name，say} from "./app.js" export default则是 import name from "./app.js"

    export default function (){ //可以没有名字 return data }

    //也可以直接一个一个的export但是必须得有名字 export const a=1； export function data(){ 　　return data; }
